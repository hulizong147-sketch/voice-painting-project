# AI 语音绘图工具设计文档

## 1. 项目背景

题目要求开发一款纯语音控制的绘图工具，用户不能依赖鼠标或键盘完成绘图创作，需要通过语音指令完成绘制、修改、选择、布局、撤销、导出等操作。同时需要考虑指令理解的准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令的拆解与执行能力。

本项目最终实现为 VoiceDraw，一个浏览器端 AI 语音绘图工作台。它支持语音绘制基础图形、设置绘图状态、生成 AI 草稿、执行局部修改、撤销重做和导出。

## 2. 设计目标

### 2.1 核心目标

1. 用户通过语音完成绘制、选择、编辑和导出。
2. 用户通过语音调整当前颜色、画笔粗细和画风。
3. 用户可以用一句复杂语音触发多个步骤，系统自动拆解并顺序执行。
4. 系统能生成复杂 AI 图片，例如动物、人物、二次元头像。
5. AI 生成结果与顶部状态一致，例如当前颜色为棕色时，生成主体应大面积使用棕色。
6. 系统提供文字和语音反馈，降低用户不知道命令是否成功的困惑。

### 2.2 体验目标

- 普通图形和编辑命令尽量即时响应。
- AI 生图允许等待，但必须显示“正在生成”反馈。
- 顶部状态栏只展示关键状态，避免调试信息堆叠。
- 颜色、画笔、画风等状态必须通过语音调整，符合题目“语音控制”的要求。
- AI 图片默认放在画布中心，便于演示和录屏。

## 3. 总体架构

```text
用户语音
  -> 浏览器麦克风采集
  -> 百度 ASR / 浏览器 Web Speech fallback
  -> 本地中文 NLU 解析
  -> DrawingCommand 结构化指令
  -> Fabric.js 画布执行器
  -> 画布更新 + 顶部状态更新 + 右侧反馈 + TTS 播报
```

## 4. 技术方案

### 4.1 前端与画布

- React + TypeScript + Vite 构建前端。
- Fabric.js 管理画布对象、图片、路径、选择、撤销重做和导出。
- Zustand 保存绘图上下文，包括当前颜色、画笔粗细、画风、选中数量、缩放比例、语音状态等。
- 顶部状态栏只显示：当前颜色、画笔粗细、当前画风、选中数量、缩放比例。

### 4.2 语音识别与反馈

- 优先使用百度短语音识别。
- 浏览器不支持或百度不可用时，使用 Web Speech API fallback。
- 支持持续监听和按住说话两种模式。
- 命令执行后通过百度 TTS 或浏览器 SpeechSynthesis 播报结果。

### 4.3 指令理解

系统使用本地规则 NLU，将语音文本转换为 `DrawingCommand`。本地规则的优点是延迟低、行为可控、适合 demo；缺点是泛化能力不如大模型。

当前解析能力包括：

- 形状：圆、矩形、三角形、线、星星、文字。
- 颜色：红、蓝、绿、黄、黑、白、灰、紫、粉、橙、棕等。
- 数量：一个、两个、三个、数字数量。
- 位置：左上、右上、左边、右边、中间等。
- 编辑：选择、移动、缩放、旋转、翻转、删除、复制、粘贴、对齐、分布。
- 状态：颜色、画笔粗细、透明度、画风、网格、缩放。
- AI：普通 AI 绘图、二次元人物、动物、局部添加元素。
- 多步拆解：按“然后、接着、之后、并且、同时、逗号、分号”等连接词拆分。

### 4.4 复杂指令拆解

复杂指令由 `parseCommands()` 完成拆解。它会把一句包含多个动作的语音切分为多个片段，再分别调用单条命令解析器，最后由画布执行器按顺序执行。

示例：

```text
画一个红色的圆，然后画三个蓝色三角形排成一排
```

拆解结果：

1. 画一个红色的圆。
2. 画三个蓝色三角形排成一排。

示例：

```text
切换成水墨画风，然后换成棕色，然后画一个松鼠
```

拆解结果：

1. 设置画风为水墨。
2. 设置当前颜色为棕色。
3. 生成一张受水墨画风和棕色状态影响的松鼠 AI 草稿。

## 5. AI 绘图设计

### 5.1 AI 草稿生成

项目接入 Right Code Draw API 作为主要 AI 生图接口，OpenAI Images API 作为备用接口。用户说“画一个松鼠”“画一个二次元少女头像”等复杂对象时，系统会生成 AI 图片并放入画布。

当前设计取舍：

- AI 图片默认放在画布中心。
- 顶部状态会影响 AI prompt。
- 当前颜色为非黑色时，prompt 会要求主体大面积使用当前颜色。
- 当前画风会转成“二次元风格、水墨风格、简笔风格”等提示。
- 当前画笔粗细会转成“细线条、中等线条、粗线条”等提示。

### 5.2 状态联动

示例操作：

```text
切换成水墨画风
换成棕色
画笔粗细 6
画一个松鼠
```

AI prompt 会被增强为类似：

```text
松鼠，水墨风格彩色草图，主体以棕色为主色，主色覆盖主体大部分区域，动物皮毛大面积使用棕色填色，粗线条，白色背景，主体清晰
```

这样可以让 AI 生成结果与顶部状态更一致。

### 5.3 局部修改

对“给它戴帽子、给它加胡须”等命令，项目不再整图重绘，而是在原图上叠加独立图层。这样可以避免 AI 重绘导致原图主体发生大变化。

限制：

- AI 图片本身是一整张位图，无法精确选择“皮毛、眼睛、尾巴”等内部区域。
- 因此“把松鼠皮毛改成棕色”更适合通过“换成棕色，然后重新生成一只松鼠”实现。

## 6. 计划支持与最终实现

| 能力类别 | 计划支持 | 当前实现 |
| --- | --- | --- |
| 语音识别 | 识别中文绘图口令 | 百度 ASR + 浏览器 fallback |
| 基础绘图 | 圆、矩形、三角形、线、星星、文字 | 已实现 |
| 状态控制 | 颜色、画笔、透明度、画风 | 已实现，主要通过语音调整 |
| 多对象绘制 | 一次绘制多个对象 | 已实现，如“三个三角形排成一排” |
| 对象选择 | 按颜色、形状、位置选择 | 已实现基础筛选 |
| 对象编辑 | 移动、缩放、旋转、删除等 | 已实现 |
| 复杂指令 | 拆解并顺序执行 | 已实现连接词拆解 |
| AI 绘图 | 生成复杂对象 | 已实现，默认放画布中心 |
| 状态影响 AI | AI 图跟随颜色、画笔、画风 | 已实现 prompt 增强 |
| 局部修改 | 在原图上加元素 | 已实现叠加帽子、胡须等 |
| 导出 | PNG、SVG、JSON | PNG/SVG/JSON 能力保留 |

## 7. 未完成部分及原因

| 未完成项 | 原因 |
| --- | --- |
| 完全零鼠标交互 | 浏览器麦克风授权、文件下载等操作受安全策略限制，需要用户确认。 |
| 通用大模型 NLU | 当前优先保证低延迟和确定性，因此使用本地规则；大模型 NLU 会增加网络延迟和不可控性。 |
| 精确局部改色 | AI 生成图是位图，不具备可编辑语义分层；要精确改皮毛、眼睛等区域需要分割或 inpainting。 |
| 高质量局部 inpainting | 当前接入接口更适合整图生成，缺少稳定的 mask 编辑流程。 |
| 离线语音识别 | Whisper 等离线模型体积和浏览器推理成本较高，暂未集成。 |
| 自动性能 benchmark | 已做体验优化，但没有完整自动化延迟测试脚本。 |

## 8. Demo 演示建议

推荐演示顺序：

1. 展示 UI：顶部状态栏、画布、右侧监听与命令历史。
2. 展示语音设置状态：“切换成水墨画风”“换成棕色”“画笔粗细 6”。
3. 展示 AI 生成：“画一个松鼠”，说明 AI 默认放画布中心，并跟随状态生成。
4. 展示复杂指令拆解：“画一个红色的圆，然后画三个蓝色三角形排成一排”。
5. 展示选择和编辑：“选中红色的圆，然后向右移动一点，然后放大一点”。
6. 展示局部修改：“给它戴帽子”，说明不重绘原图，只叠加局部图层。
7. 展示撤销、重做和导出。

桌面上的 `VoiceDraw_Demo演示流程.md` 提供了完整逐字录制稿。
